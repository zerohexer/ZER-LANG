Comprehensive parallel agent audit of the ZER compiler (`/home/user/ZER-LANG`)
on branch `claude/cool-johnson-5v2OE`. Five specialized agents ran in parallel,
each reading 100K+ tokens of source and verifying findings with the built

**Scope ignored:** `zercheck.c` (now a 150-line shim — production path is
`zercheck_ir.c` per CLAUDE.md "CFG Migration COMPLETE 2026-05-03").

## Bugs FIXED in this session

### BUG-643 — CRITICAL silent baremetal MMIO miscompile (orchestrator find)

`volatile *u32 reg = @inttoptr(*u32, ADDR);` at local scope emitted plain
`uint32_t* reg = ...` in C. The `volatile` qualifier was carried only on
`Symbol.is_volatile` and `var_decl.is_volatile`, never propagated into the
`Type *` returned by `checker_get_type()`. IR lowering (`ir_lower.c:1695`)
read the unqualified type, the emitter then emitted without `volatile`,
and **GCC -O2 collapsed sequential register writes to a single store.**

Proof (before fix, x86-64 -O2 asm):
```
mmio_write_seq:
    movl $0xcafe, 0x40000000   # 3 writes collapsed to 1
    ret
```

After fix:
```
mmio_write_seq:
    movl $0xdead, 0x40000000   # all 3 preserved
    movl $0xbeef, 0x40000000
    movl $0xcafe, 0x40000000
    ret
```

**Impact**: every embedded program using `volatile *u32 reg = @inttoptr(...)`
for MMIO at function scope. Breaks UART transmit sequences, timer
reset/enable, DMA descriptor setup, interrupt clear sequences. The bug
existed because `volatile` was applied to the VARIABLE qualifier (like
`const`), not threaded into the pointer Type. Global var-decls (AST path)
had the propagation at `emitter.c:3795`; locals (IR path) did not.

**Fix**: `checker.c:7608-7613` — extend the existing TYPE_SLICE volatile
propagation to also cover TYPE_POINTER (with TYPE_DISTINCT unwrap).
Mirrors `emitter.c:3795` so AST and IR paths converge.

**Test**: `tests/zer/_verify_BUG-643_volatile_local_ptr.zer`. Manual
verification: `--emit-c` + `gcc -O2 -S` + `objdump -d` shows 3 distinct
stores.

### BUG-644 — Comptime depth counter leak (orchestrator find)

`eval_comptime_call_subst` in `checker.c:1757` incremented
`_comptime_call_depth` at line 1770 but RETURNED `CONST_EVAL_FAIL` at
lines 1773 and 1776 without decrementing. After ~17 such leaks (caused
by comptime function calling a non-comptime function, or with wrong
arity), the next legitimate comptime call falsely tripped the
depth-exceeded guard:

```
error: comptime call chain exceeded recursion depth (16)
```

**Fix**: add explicit `_comptime_call_depth--;` before each early
return. Same pattern as `Plan` mentioned in CLAUDE.md "Anti-Patterns
Found in This Session" (save/restore pattern).

**Test**: `tests/zer/_verify_BUG-644_comptime_depth_leak.zer`.

## Bugs FOUND — needs design decision before fixing

### From Agent 1 (IR-path safety-wrapper diff audit)

1. **Slice constructor `arr[a..b]` with b > arr.len silently accepted**.
   Both AST and IR paths only check `start > end`, never the upper bound
   against `arr.len`. Reproducer compiles + runs returning slice.len=100
   from a 4-element array — subsequent indexing reads adjacent stack.
   Verified with `/tmp/verify_slice_oob.zer`. Gap test:
   `tests/zer_gaps/audit_2026-06-02_slice_oob.zer`.

2. **IR_BINOP `/` and `%` defense-in-depth gap**. Checker forces compile-
   time div-zero proof, so today nothing slips through. But the IR emitter
   at `emitter.c:9404-9425` does not emit the trap that the AST path emits.
   If any future VRP improvement creates a "may-be-zero" function-call
   chain that bypasses the checker, the IR path will miscompile silently.
   Recommend porting the wrapper for symmetry.

3. **IR_CAST `@saturate(i64, ...)` emits malformed integer literal**. IR
   path computes `1LL << 63`. GCC accepts with warning; clang/MSVC reject.
   AST path correctly shortcuts.

4. **@ptrcast between two unrelated CONCRETE struct pointers silently
   allowed**. Provenance tracking only fires when one side is `*opaque`.
   `@ptrcast(*B, &a)` of plain `*A` falls through to a plain C cast.
   Reproducer returns 100 (low byte of `A.x` as `B.y`). Gap test:
   `tests/zer_gaps/audit_2026-06-02_ptrcast_unrelated.zer`.

5. **Dead-stub opcodes**. `IR_INDEX_WRITE` / `IR_ADDR_OF` / `IR_DEREF_READ`
   / `IR_CALL_DECOMP` / `IR_INTRINSIC_DECOMP` / `IR_ORELSE_DECOMP` /
   `IR_SLICE_READ` in `emitter.c:9677-9688` emit `/* 3AC op N — TODO */`.
   Lowerer never produces them today. Recommend `abort()` or `#error` so
   accidental emission becomes a hard failure (matches CLAUDE.md
   "Dead-stub pattern" prevention rule).

### From Agent 2 (zercheck_ir handle/state-machine)

1. **Orelse-reassignment overwrites alive handle silently** (Model 1).
   `h = pool.alloc() orelse return;` decomposes to IR_COPY, but
   IR_COPY's overwrite-leak check is missing. The first allocation is
   leaked silently. F3.1 partial fix didn't reach this decomposed form.

2. **Global handle alias UAF** (Model 2). `g_stored = h; pool.free(h);
   pool.get(g_stored)` — `ir_target_root_escapes` marks `h.escaped=true`
   to suppress leak but doesn't register `g_stored` as compound key.
   Subsequent get is unchecked.

3. **defer free + manual free** (Model 1). Sequential `defer p.free(h);
   p.free(h);` compiles clean, double-frees at runtime.

4. **Variable-index array element re-access after free** (Model 2).
   `ir_extract_compound_key` only accepts NODE_INT_LIT indexes — `arr[k]`
   with non-literal `k` fails key extraction.

5. **Handle stored in array element never tracked for leak** (Model 1).
   `arr[0] = pool.alloc(); arr[1] = pool.alloc(); p.free(arr[0]);` — arr[1]
   leaks silently.

6. **Cross-function wrong-pool detection missing** (Model 3). F3.2 added
   intra-function `pool_name` tracking but no `FuncSummary` field.

7. **Use-after-move via pointer dereference read** (Model 1). `File copy
   = *p;` doesn't transfer ownership of underlying f.

8. Bonus — Optional-handle field re-unwrap after free (Model 2).

### From Agent 3 (baremetal/hardware)

Verified by orchestrator:

1. **`--no-strict-mmio` drops the RUNTIME range/alignment check too**.
   Per CLAUDE.md it should only relax the compile-time requirement.
   Variable-address `@inttoptr` with `--no-strict-mmio` emits raw
   `((uint32_t*)(uintptr_t)(addr))` with no check. Gap test:
   `tests/zer_gaps/audit_2026-06-02_nostrict_mmio_no_runtime.zer`.

2. **`shared struct` accessed from `interrupt` deadlocks**. Emitter
   inserts `pthread_mutex_lock` inside `USART1_IRQHandler`. ISR firing
   while main holds the lock = deadlock. Should be banned per
   CLAUDE.md ban framework (Hardware/OS Constraint test).

3. **`@cpu_disable_int/wait_int/enable_int` emit unguarded** when called
   from non-naked, non-interrupt context. Dead-branch convention is
   documentation only, not enforced.

4. **`@ptrcast(*u32, &buf[1])` accepts misaligned source** (silent
   BusFault on Cortex-M0/M0+).

5. **`@cpu_save_context(*u8)` accepts undersized buffer** (CLAUDE.md
   mandates 128+; no check for `u8[16]`).

6. **Global `Pool(T, N)` is NOT atomic**. ISR preempts allocator
   between `if(!used[i])` and `used[i]=1` → both contexts get same
   handle. CLAUDE.md positions Pool as "ISR-safe alternative to Slab";
   in fact Pool's RMW is non-atomic.

7. **`Ring(*T, N)` is warning, not error**. Raw pointer through channel
   = silent UAF in receiver.

8. **`@inttoptr(*u32, 0x0)` accepted** when range covers it.
   Overwriting NMI vector table possible if user declares wide range.

9. **`@probe(addr)` is unsafe on freestanding**. Hosted branch uses
   setjmp/SIGSEGV; freestanding emits raw `*(volatile uint32_t*)addr`
   — opposite of its documented contract ("returns null if address
   faults"). HardFaults instead of returning null.

10. **Stack-touching asm in `naked` not detected**. `asm("push %rax")`
    accepted; CLAUDE.md S1 documents this as interim.

2. **Range-for over `shared` struct fixed array emits `array.len`**
   (invalid C). Field access `g.data` is `uint32_t[8]`, not slice.

3. **Async + explicit return + defer emits defer body twice** (currently
   dead code at function tail — fragile).

4. **`slab.alloc(T)` rejected** with terse error; `pool.alloc(T)` accepted.
   Spec/UX inconsistency.

5. **Ghost-handle false positive when result captured in `?Handle`**.

6. **33 of 88 extracted safety predicates are orphaned** (no call sites).
   `src/safety/*.c` has 88 functions; only ~55 are referenced in the
   compiler. Level 3 VST proofs cover code that doesn't make decisions —
   matches the "frozen-spec" anti-pattern CLAUDE.md warns about.

7. **Fixed `char[256]/[512]` buffers** for mangled identifiers at 6 sites
   (`checker.c:1583/7144/10926/10944/10961`, `emitter.c:9919`). Silent
   truncation if names exceed.

8. **Static-int recursion counters not thread-safe and leak on
   early-exit paths** (5 sites — `_container_depth`, `_subst_depth`,
   `eval_comptime_block depth`, `_scan_depth`, plus the
   `_comptime_call_depth` we just fixed). Same root cause as BUG-644.

9. **Module-name mangling duplicated 4+ times** without a helper.

### From Agent 5 (IR validation + walkers)

1. **`IR_YIELD/IR_AWAIT` outside async function** — not validated by
   `ir_validate`. Lowerer bug → broken Duff's device C.

2. **`IR_LITERAL.literal_kind` not range-checked**. Emitter switch has
   no default; out-of-range value emits `= ;` (C syntax error in
   output, not source).

3. **`ir_compute_preds` and `ir_block_is_terminated` treat `IR_AWAIT`
   as non-terminator**. But `cfg_reaches_fire` does treat it specially.
   Fragile invariant — depends on no `ir_add_block` between
   `IR_AWAIT` and `current_block` reassignment.

4. **Emitter dead stubs for unused 3AC ops** (same as Agent 1 #5 above).

5. **`IR_FIELD_WRITE` falls through into `IR_CAST` handler**.
   `zercheck_ir.c` has a dedicated `IR_FIELD_WRITE` analyzer but the
   emitter would silently emit nothing if it were ever reached.

6. **`IR_SPAWN` move-struct transfer is too narrow**. Compare
   `IR_CALL`: walks NODE_FIELD/NODE_INDEX to root, unwraps
   NODE_UNARY/AMP. `IR_SPAWN` only catches bare NODE_IDENT args.
   Verified silent gap.
